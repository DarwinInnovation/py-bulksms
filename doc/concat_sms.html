NOKIA 9000i COMMUNICATOR

AND

NOKIA 9110 COMMUNICATOR

CONCATENATED SHORT MESSAGES

  1. Background

In the GSM network there is a possibility to send short messages, up to 160 characters long. The GSM specification version 03.40 defines concatenated short messages which can be even longer. The Nokia 9000i Communicator and Nokia 9110 Communicator support concatenated short messages up to 2280 characters.

  1. Definition

With concatenated short messages it is necessary to use a certain header which defines the message so that several messages can be sewed into one. In practise the concatenated short message concludes of a header of 56 bits and maximum of 152 7 bit characters. The header is as follows:

1st byte

2nd byte

3rd byte

4th byte

5th byte

6th byte

7th byte

Length of the header

IEI

Length of the sub-header

Reference number

Maximum amount of pieces

Sequence number

Fill

IEI = Information Element Identifier.

The first byte value is 5, defining the length of the whole header. IEI is set to 0 in the second byte, 0 is the identification number for concatenated SMS. In the third byte the length of message definition part of the header is defined. Fourth byte is the Reference number for the whole concatenated message. The device identifies the message by checking that the phone number of the message sender and the Reference number are the same with each part of the message. The Reference number in the communicator is a running number, where one is added to each concatenated message to separate them. The fifth byte defines the number of pieces in the whole message and the sixth byte defines the number of the piece.

The header information is in 8 bit format. However the communicator sends the messages in 7 bit format. The fill is needed at the end of the header, to make the data start from the first whole 7 bit byte. (Because: 160 * 7 bits = 1120 bits - (6 * 8 bits) = 1072 bits / 7 bits = 153,143 7 bit characters which is not good. So to make the header one 8 bit byte longer: 1120 bits - (7 * 8 bits) = 1064 bits / 7 bits = 152 7 bit characters.)

When sending a concatenated message the UDHI bit is set, as defined in the GSM 03.40 specification. But when receiving the message the Nokia 9000i Communicator and the Nokia 9110 Communicator do not require that bit to be set, contrary to the GSM 03.40 specification. This is due to the fact that the Short Message Service Centres in current neworks do not usually keep the UDHI bit set.
